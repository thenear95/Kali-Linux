Attaque WPA2 ou Attaque KRACK

Récemment, Mathy Vanhoef d'imec-DistriNet, KU Leuven, a découvert une faiblesse sérieuse dans WPA2 connue sous le nom d'attaque AttaCK (ou KRACK) de réinstallation de clé. Leur vue d'ensemble, les attaques de réinstallation clés: forcer la réutilisation de nonce, et le document de recherche (Attaques clés de réinstallation: Forcer Nonce réutilisation dans WPA2, co-écrit par Frank Piessens) ont fait beaucoup de bruit dans notre industrie. De nombreux articles ont été écrits sur cette vulnérabilité, et nous ne les reprendrons pas ici. Cependant, nous voulons prendre un moment pour parler de ce qui se rapporte à Kali Linux, d'un point de vue défensif, de test et de détection..

Kali Linux est-il vulnérable ??

D'un point de vue défensif, si vous suivez MAJ roulantes de Kali Linux (via une simple mise à jour d'apt && apt), vous avez déjà été corrigé de cette vulnérabilité grâce aux correctifs de wpa supplicant et hostapd (tous deux à 2.4-1.1 ). Pour être tout à fait clair: une version mise à jour de Kali Linux n'est pas vulnérable à cette attaque.

Comment tester si nous sommes vulnérables ?

Avec votre système Kali mis à jour, vous pouvez également suivre certaines étapes pour tester cette vulnérabilité sur vos points d'accès. Mathy Vanhoef a récemment publié un script qui peut être exécuté à partir de Kali Linux pour tester si votre point d'accès (AP) est affecté par CVE-2017-13082 ou plus précisément la vulnérabilité Key Reinstall in FT Handshake trouvée dans les périphériques 802.11r. Le script nécessite que vous vous authentifiez auprès du point d'accès, mais gardez à l'esprit qu'il peut signaler de manière incorrecte un point d'accès comme vulnérable en raison de «retransmissions bénignes de trames de données».

Comment détecter les attaques ?

Dragorn, l'auteur de l'incroyable Kismet, a publié beaucoup d'excellentes informations sur le sujet sur son blog, y compris d'excellentes informations sur la détection des attaques KRACK en utilisant Kismet. Il explique que la version git-master de Kismet est "l'introduction d'alertes pour tenter de détecter une attaque de type "craquage".

Kali Linux 2017.3 dévoilé

Kali Linux 2017.3, est dispo il inclut tous les correctifs, correctifs, mises à jour et améliorations depuis notre dernière version. Dans cette version, le noyau a été mis à jour en version 4.13.10 et inclut des améliorations notables:

  • CIFS utilise SMB 3.0 pr défaut

  • EXT4 les répertoires peuvent maintenant contenir 2 milliards d'entrées au lieu de l'ancienne limite de 10 millions

  • TLS le support est maintenant intégré dans le noyau lui-même

En plus du nouveau noyau et de toutes les mises à jour et correctifs que nous avons tirés de Debian, nous avons également mis à jour nos paquets pour Reaver, PixieWPS, Burp Suite, Coucou, la boîte à outils d'ingénierie sociale, etc. Jetez un coup d'oeil sur le Kali Changelog pour voir ce qui a été mis à jour dans cette version, ou lisez ce qui suit pour voir ce qui est nouveau.
